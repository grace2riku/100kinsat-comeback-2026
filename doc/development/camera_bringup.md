# カメラ（Spresense カメラボード）赤コーン検出 立ち上げ・実機確認手順（Issue #52）

ホストテストで確認できるのは「**HW非依存ロジック**」だけ（gotchas A）。具体的には合成 UYVY 画像に
対する赤閾値判定・列ヒストグラム・区間連結・重心/幅/方位角の算出（`test/core/test_cone_detect.cpp`）。
次の項目は実機でしか確認できない:

- **カメラボードで実際に撮像できる**（接続・ドライバ初期化・メモリ確保）
- **YUV422 のバイト順が UYVY である**（core の前提。取り違えると検出全滅）
- **実物の赤コーンが屋外の光条件で検出できる**（色閾値の物理的妥当性）
- **撮像＋検出の1サイクル処理時間**（Phase3 制御周期との整合）
- **水平画角（方位角換算の校正）**

本書はその確認手順と記録テンプレート。確認結果を埋めてコミットし、Issue #52 の実機 DoD を
満たしたうえでマージする（HWモジュールのマージゲート: 実機確認まで Issue をクローズしない）。

対象コマンド: シェル（`src/shell`）の `cam <init|snap|dump|detect|mon [n]|thr [yMin yMax uMax vMin]>`
検出ロジックは `src/lib/core/cone_detect`、撮像・SD保存は `src/lib/hal/spresense_camera.h` /
`src/lib/hal/sd_image_store.h`（HAL 薄呼び出し）。

---

## 0. ⚠️ 注意・前提

- **カメラボードはメインボードの専用コネクタ（CIF）に接続**する。フレキシブルケーブルの向き・
  ロックを確認（hardware.md）。既存ピンアサイン（モータ/I2C/D06/D09 等）とは競合しない。
- **`cam init` はオンデマンド初期化**（gotchas B9: 起動経路にデバイス初期化を置かない）。
  初期化はドライバ起動＋ビデオバッファ約150KB（QVGA/YUV422）の確保を伴う。
- **ホワイトバランスは DAYLIGHT 固定**（`cam init` が設定）。AWB のままだと至近距離でコーンが
  画面を占めたとき赤に引っ張られて色相が回り、閾値が安定しない恐れがあるため。
- **SD カード（FAT32）が必要**なのは `cam snap` / `cam dump` のみ。detect/mon は SD 不要。
- 色閾値の既定（yMin=32 yMax=235 uMax=120 vMin=160）は「彩度の高い赤」の机上初期値。
  **屋外で必ず `cam thr` により調整**し、採用値を本書 §3 に記録する。
- 水平画角の既定 66° は カメラボード仕様（レンズ画角 78°±3°（対角）・EFL 2.74mm・1/4型センサ）
  からの机上見積り。方位角を Phase3 の操舵に使う前に §2 手順E で実測校正する。

## 1. 準備

1. シェルをビルド＆書き込み:
   ```bash
   tools/build.sh  src/shell
   tools/upload.sh src/shell
   ```
2. シリアルモニタを開く（115200bps）。プロンプト `100kinsat>` を確認。
3. `cam init` を実行し「初期化完了」が出ること。カメラ未接続時は
   `初期化失敗 (NO_DEVICE)` 等のエラー名が出て**シェルが固まらない**ことも確認する。

## 2. 確認手順

### 手順A: 撮像・SD 保存（実機 DoD「撮像・SD保存が動作」）

```
cam init
cam snap        （VGA JPEG → cam/imgNNN.jpg）
cam dump        （QVGA YUV422(UYVY) 生データ → cam/imgNNN.yuv, 153,600 bytes）
```

SD を PC で開き確認:
- `img*.jpg` が画像ビューアで正常に見える（構図・露出）
- `img*.yuv` を変換して正常に見える（**UYVY バイト順の実機確認**）:
  ```bash
  ffmpeg -f rawvideo -pix_fmt uyvy422 -video_size 320x240 -i img000.yuv -update 1 -frames:v 1 img000.png
  ```
  （`-update 1 -frames:v 1` は単発画像出力の指定。無くても PNG は生成されるが警告が出る）
  色が正常なら UYVY 前提は正しい。**緑/紫に化けたらバイト順が異なる**ので
  `cone_detect.h` の前提を修正すること（gotchas A5: フォーマット前提は実機で突き合わせ）。

### 手順B: 赤コーン検出（距離別）

赤コーン（実物または同等品: 高さ70cm・幅37.5cm・赤色無地）を屋外に置き、
距離 1 / 3 / 5 / 10 m から:

```
cam detect      （単発: detected / bearing / width / ratio / conf / pixels と処理時間）
cam mon 20      （連続20回: 検出の安定性。非改行キーで中断）
cam dump        （各距離で実写ダンプも保存 → §4 の体系保存へ）
```

観察ポイント:
- 各距離で `detected=1` になるか（10m で width ≈ 10 col 程度の見込み）
- コーンを画角の左/中央/右に置いたとき `bearing` の符号・大きさが妥当か
- コーン無しの背景で `detected=0` か（誤検出の有無）

### 手順C: 光条件別の閾値調整（順光/逆光/日陰）

同一距離（例 3m）で順光・逆光・日陰それぞれ:

```
cam detect      （既定閾値での検出可否）
cam thr <yMin> <yMax> <uMax> <vMin>   （未検出/誤検出があれば調整）
cam dump        （各条件の実写ダンプを保存）
```

- 逆光では輝度が落ち Cr が下がりやすい → `vMin` を下げる方向。ただし下げすぎると
  茶色・肌色を誤検出するので、誤検出が出ない範囲で調整する
- 調整後の採用値と条件別の傾向を §3 に記録する

### 手順D: 処理時間の実測（Phase3 制御周期との整合）

`cam detect` の表示（`capture=...ms detect=...ms`）を10回程度記録し、
撮像＋検出の1サイクル時間の代表値を §3 に記録する。Phase3（#17/#18）の制御周期設計
（モータ制御・IMU 読みとの共存）への入力になる。検出フレームはビデオストリーミング
（5fps）経由なので capture の待ちは最大でフレーム間隔（約200ms）程度になる想定。
周期が足りない場合は begin() の fps 引数を上げる判断を #18 へ申し送る。

### 手順E: 水平画角の校正（方位角の妥当性）

1. 機体を固定し、正面（bearing≈0）にコーンを置いて `cam detect` で bearing≈0 を確認。
2. コーンを真横に既知距離だけずらす（例: 距離 3m で横 1m → 実角度 atan(1/3)=18.4°）。
3. `cam detect` の bearing と実角度を比較し、ずれる場合は `cone::Config::hfovDeg` を
   実測に合わせて修正（ソース既定値の更新）し、採用値を §3 に記録する。

### 手順F: メモリ共存（MemSize）

`cam init` 後に既存機能と共存できることを確認する（カメラのバッファ確保が他機能の
メモリを圧迫しないこと）:

```
cam init → imu init → gnss init → cam detect → log 5 → cam snap
```

いずれもエラーなく動作すること（`NO_MEMORY` が出る場合はビデオバッファ数・解像度の
見直し、または MemSize 設定の変更を検討し、結果を gotchas へ資産化する）。

### 実機確認結果（一部実施: 2026-07-19 手順A）

- `cam init` / `cam snap` は成功（`cam/img000.jpg` 27,200 bytes が期待どおりの画像）。
- 初版実装の `cam dump`（**still 撮影を QVGA/YUV422 で行う方式**）は
  `setStillPictureImageFormat` が SUCCESS を返すのに `takePicture()` が失敗し再現性あり
  （JPEG still は成功するため、**ISX012 は still=JPEG のみ実用**と判断。gotchas B23）。
  → 検出用 YUV フレームを**ビデオストリーミング経由**で取得する方式へ設計変更した。
- **設計変更後ファーム（2507d96）で手順Aを再確認**: `cam init` / `cam snap`
  （`cam/img001.jpg` 26,720 bytes・拡張子別の独立連番も期待どおり）/ `cam dump`
  （`cam/img000.yuv` **153,600 bytes = QVGA UYVY の期待サイズと一致**）すべて成功。
  ffmpeg（uyvy422 指定）で PNG 変換し、**snap の JPEG・dump の変換画像とも色が自然で
  あることを目視確認 → UYVY バイト順（core cone_detect の前提）は実機で正しいと確定**。
- 未実施の残項目: 未接続時の `cam init` エラー挙動（手順A）、手順B〜F、§4 の体系保存。

## 5. 申し送り（#17/#18/#19 結合 / #29 精度）

- **#18（自律走行）**: 検出 I/F（`cone::Detection` の bearing/widthRatio）を終端誘導の
  操舵入力へ結線。処理時間（手順D）を制御周期設計に反映。周期不足ならストリーミング化を検討。
- **#19（ゴール判定）**: `widthRatio` のしきい値（例: 至近 0.5m で ≈0.6）で停止判定。
  実測データ（手順B の距離別 width）から決める。
- **#29（Phase5）**: 実写データ蓄積後、誤検出対策（形状検証・DNNRT 検証器）の要否を判断。
