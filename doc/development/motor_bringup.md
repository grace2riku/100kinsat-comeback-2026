# モータ立ち上げ・実機検証手順（TB6612FNG / Issue #8）

ホストテストでは「真理値表どおりにピンへ出力されること」までしか確認できない（gotchas A）。
**物理的な前進/旋回方向**と**最低駆動デューティ（デッドゾーン）**は実機で確認する必要がある。
本書はその計測手順と記録テンプレートを兼ねる。計測値を本書の表に記入してコミットし、Issue #8 の
再導入PRに添える（DoD「実機でデッドゾーン/最低駆動デューティを計測し記録」を満たす）。

対象コマンド: シェル（`src/shell`）の `motor <forward|back|left|right|stop> [duty 0-255] [ms]`
ピン割当（暫定）: `src/lib/hal/spresense_pins.h` … 左=モータA`{D08,D04,D05}` / 右=モータB`{D07,D02,D03}`

---

## 0. ⚠️ 安全上の注意（最重要・必ず守る）

- **車輪を地面に着けない。** 機体を台や箱の上に乗せ、**車輪を浮かせた状態**で行う（暴走して走り出さないため）。
- **低 duty・短時間から始める。** 最初は `duty=10〜20`, `ms=500` 程度。いきなり全力(255)で回さない。
- **緊急停止**: コマンドは `ms` 経過後に自動停止する（上限5000ms）。それでも止まらない/異常時は**モータ電源を切る**。
  - 注意: `motor <dir>` の駆動中はシェルが `delay` でブロックし `stop` を受理しない。だから低 duty・短 ms で刻む。
- **指・髪・ケーブルを車輪/ギヤに近づけない。**

## 1. 準備

1. **モータ電源(6V)を接続**する（hardware.md: モータドライバへ 6V を直接供給）。
   - これが無いとモータは回らず、「デッドゾーンが高い」と誤判定する。`STBY` は基板で常時HIGH（操作不要）。
2. シェルをビルド＆書き込み:
   ```bash
   tools/build.sh  src/shell
   tools/upload.sh src/shell
   ```
3. シリアルモニタを開く（115200bps）。書き込み直後はバナーが流れるので、**モニタを開いてからボードをリセット**するとプロンプト `100kinsat>` が見える。
4. `help` で `motor` コマンドの書式を確認。

## 2. 手順A: 最低駆動デューティ / デッドゾーン計測

両モータを前進方向に、duty を低い方から少しずつ上げ、**各車輪が回り始める duty** を観察する。
（両輪を同時に駆動するが、各輪はそれぞれの最低 duty を超えた時点で回り始めるので、左右別々に観察できる。）

操作例（duty を刻んで投入。各回 800ms 回って自動停止）:
```
motor forward 10
motor forward 20
motor forward 30
...
```

## 3. 手順B: 物理方向の確認

各コマンドを低 duty（例 80〜120）で投入し、**機体（または車輪）の実際の動き**を観察する。

### 記録テーブル

| コマンド | 期待する動き | 実際の動き | 判定 | 対処（4章） |
|---|---|---|---|---|
| `motor forward 100` | 機体が**前進** |  | OK / 反転 / スピン |  |
| `motor back 100`    | 機体が**後進** |  | OK / 反転 |  |
| `motor right 100`   | **右**ピボット旋回（左輪前進・右輪後進） |  | OK / 反転 |  |
| `motor left 100`    | **左**ピボット旋回（左輪後進・右輪前進） |  | OK / 反転 |  |

判定の見方:
- **OK**: 期待どおり。
- **反転（前後）**: `forward` で後ろへ → 前後が逆。
- **反転（旋回）**: `right` で左に回る → 左右の割当が逆。
- **スピン**: `forward` で機体がその場回転（片輪前・片輪後）→ 左右モータがミラー取付で極性が逆。

## 4. 方向がずれていたときの調整（`src/lib/hal/spresense_pins.h` を編集）

物理方向のずれは**配線ロジック側のピン割当だけ**で直せる（モータ制御ロジック `motor.cpp` は変更不要）。
`Pins{in1, in2, pwm}` の並びを編集する。編集後は `tools/build.sh src/shell && tools/upload.sh src/shell` で再確認。

| 症状 | 原因 | 対処（spresense_pins.h） |
|---|---|---|
| **前後が逆**（forward で後進） | 両モータの極性が物理と逆 | 両モータの **in1 と in2 を入れ替える**（例: 左 `{8,4,5}`→`{4,8,5}`、右 `{7,2,3}`→`{2,7,3}`） |
| **旋回が左右逆**（right で左旋回） | 左右の割当が逆 | **`kMotorLeft` と `kMotorRight` を入れ替える** |
| **forward でスピン**（片輪前・片輪後） | 左右モータがミラー取付で極性が逆 | **片側のモータだけ in1/in2 を入れ替える**（例: 右だけ `{7,2,3}`→`{2,7,3}`） |

> ホストテスト `test/core/test_motor.cpp` は「割当てられたピンに真理値表どおり出力する」ことを検証している。
> 上記はピン割当（配線の対応）の調整なので、ロジックの真理値表は不変＝ホストテストは緑のまま。
> 割当を変えた場合は、その理由を本書「調整記録」に残すこと。
