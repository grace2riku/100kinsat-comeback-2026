# 9軸センサ立ち上げ・実機検証手順（BNO055 / Issue #9）

ホストテスト（`test/core/test_compass.cpp`）で確認できるのは「方位の正規化・最短回頭角・校正完了判定」という**HW非依存ロジック**だけ（gotchas A）。次の項目は実機でしか確認できない:

- **センサの検出・I2C 初期化**（未検出時のエラーハンドリング）
- **キャリブレーションが実際に 0→3 へ収束すること**
- **方位角の物理的な正しさ**（北で 0°付近、時計回りで増加するか／磁気偏角・取付向き）
- **`isHeadingReady`（sys+mag==3）の暫定しきい値が方位精度に十分か**（gyro 未校正時の挙動）

本書はその計測手順と記録テンプレートを兼ねる。計測値を埋めてコミットし、Issue #9 のHW依存DoDを満たしたうえで再導入/マージする（HWモジュールのマージゲート: 実機確認まで Issue をクローズしない）。

対象コマンド: シェル（`src/shell`）の `imu [init|cal|mon [n]]`
結線: BNO055 を I2C で接続（`D14=SDA / D15=SCL`、アドレス `0x28`）。hardware.md / software.md §5.6 参照。

---

## 0. ⚠️ 注意

- 配線（SDA/SCL/電源/GND）を**通電前に**確認する。I2C アドレスは出荷時 `0x28`。ジャンパで `0x29` にした場合は `src/shell/shell.ino` の `static hal::Bno055Compass g_imu;` を `static hal::Bno055Compass g_imu(0x29);` に変えて再ビルドする。
- 方位は地磁気を使うため、**モータ・スピーカ・大電流配線・金属・磁石から離して**校正・計測する（磁気が乱れると方位が大きくずれる）。
- `imu mon` は読み取り専用で HW を駆動しないため、`delay` でブロックしても安全（暴走しない。gotchas B5 の駆動系コマンドとは別扱い）。

## 1. 準備

1. Adafruit ライブラリを導入済みか確認（未導入なら [build_arduino_cli.md §2.5](./build_arduino_cli.md)）:
   ```bash
   arduino-cli lib list | grep -i "Adafruit BNO055"
   ```
2. シェルをビルド＆書き込み:
   ```bash
   tools/build.sh  src/shell
   tools/upload.sh src/shell
   ```
3. シリアルモニタを開く（115200bps）。**モニタを開いてからボードをリセット**するとバナーと `imu:` の検出メッセージが見える（A2: `while(!Serial)` で待たない運用）。
4. `help` で `imu` の書式を確認。

## 2. 手順A: 検出・初期化（未検出ハンドリング）

起動時に次のどちらかが出る:
- `imu: BNO055 検出・初期化 OK`
- `imu: BNO055 未検出（…）` → 配線・電源・アドレスを確認し、`imu init` で再試行。

### 記録

| 状況 | 期待 | 実際 | 判定 |
|---|---|---|---|
| 正常配線で起動 | 「検出・初期化 OK」 |  |  |
| SDA か SCL を外して `imu init` | 「未検出」エラー（ハングしない） |  |  |
| 未検出のまま `imu`（読み取り） | 「未初期化」案内（無効値で誤動作しない） |  |  |
| 検出後に SDA か SCL を外して `imu stat` | 「異常」（sys_status≠5）。健全性は **`imu stat` で確認する** |  |  |

> 重要: 活線抜け時の `heading` は getVector の仕様上 `0.0`（=北）になり**値だけでは故障と真北を区別できない**（gotchas B8）。故障検知は必ず `imu stat`（`sys_status==5` で融合稼働中）で行う。

> 未検出時に**ハングせず・無効値で先に進まない**ことが DoD「センサ未検出時のエラーハンドリング」の確認点。

## 4. 手順C: 方位角の物理的な正しさ

校正完了後、本体の**前方（基準方向）**を実際の方位に合わせ、`imu` の `heading` を読む。スマホのコンパス等で実際の方位を確認しながら4方位で比較する。

| 本体を向けた実際の方位 | 期待 heading | 実測 heading | 差 | 備考 |
|---|---|---|---|---|
| 北 | 0°（±数°） |  |  |  |
| 東 | 90° |  |  |  |
| 南 | 180° |  |  |  |
| 西 | 270° |  |  |  |

確認ポイント:
- **時計回り（北→東→南）で heading が増加**するか（逆なら取付向き/軸の定義がコードの想定と逆）。
- **0/360 の境界**を跨いでも `imu` の表示が `[0,360)` に収まるか（正規化の実機確認）。
- **磁気偏角**: スマホの「真北」とセンサの「磁北」で数°ずれるのは正常。種子島の偏角分は後段ナビ（#18）でオフセット補正する方針か、ここに実測差を記録しておく。

## 5. 手順D: `isHeadingReady`（暫定しきい値）の妥当性

`isHeadingReady` は `sys==3 && mag==3` で「方位使用可」と判定するが、これは spec 裏付けのない暫定ヒューリスティック（compass.h 参照）。NDOF では gyro も方位ドリフト補正に効くため、**gyro 未校正のまま方位を使うと走り出しでドリフトしうる**。

### 確認

- `imu mon` で **sys=3,mag=3 だが gyro<3** の状態を作り、本体を静止させて heading のふらつき/ドリフトを観察する。
- 同条件で本体をゆっくり一定速度で回し、heading が滑らかに追従するか。

| 条件 | heading のふらつき | ドリフト | 所見 |
|---|---|---|---|
| sys=3,mag=3,gyro<3 |  |  |  |
| 全要素3 |  |  |  |
