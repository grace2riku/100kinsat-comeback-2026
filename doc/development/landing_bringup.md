# 着地(静止)検知 立ち上げ・実機検証手順（BNO055 加速度 / Issue #12）

ホストテストで確認できるのは「**HW非依存ロジック**」だけ（gotchas A）。具体的には加速度の大きさ |a| の
窓内分散＋平均から静止／着地を判定する状態機械（`test/core/test_landing.cpp`：`LandingDetector` /
`isStillNow` / `hasLanded`）と、生バイト→加速度 [m/s^2] 変換（`accelFromRaw`）。次の項目は実機でしか
確認できない:

- **BNO055 の加速度読み取り**（`readAcceleration` の自前 Wire 読みが成功し、据置で `|a|≒9.8` になるか）
- **静止→着地確定の収束**（据置で数秒後に「着地検知」へ至るか。しきい値が実機ノイズに合うか）
- **着地衝撃と静止の切り分け**（叩く/揺らすと分散が上がり静止にならず、置き直すと再確定するか）
- **読み取り堅牢性**（振動・EMI 下で読み取り失敗（I2C）を静止に流さないか）

本書はその計測手順と記録テンプレートを兼ねる。計測値を埋めてコミットし、Issue #12 のHW依存DoDを
満たしたうえでマージする（HWモジュールのマージゲート: 実機確認まで Issue をクローズしない）。

対象コマンド: シェル（`src/shell`）の `land [mon [n]]`
判定ロジックは `src/lib/core/landing`、加速度取得は `src/lib/hal` の `Bno055Compass::readAcceleration`。
加速度は BNO055 の **ACC_DATA(0x08, 重力込み)** を読む（融合モードでも生加速度。gotchas B16）。

---

## 0. ⚠️ 注意

- **加速度は `imu init`（BNO055 初期化）が前提。** `land` は `g_imu` から読むため、先に `imu init` で
  検出・初期化してから使う（未初期化なら「9軸センサ未初期化」と表示して先に進まない）。
- **`land` / `land mon` は読み取り専用で HW を駆動しない**ため安全（暴走しない。gotchas B5）。`mon` 中は
  **非改行キー入力で即中断**できる（CR/LF はコマンド確定の残留改行なので中断トリガーにしない。gotchas B10）。
- **しきい値は暫定**（`landing::defaultStillConfig()`：窓20・分散上限0.5・g許容1.5・静止1秒）。本手順で
  実機の据置ノイズ・着地衝撃の実測を取り、値を詰めて #17（ステートマシン）/#12 実利用へ申し送る。

## 1. 準備

1. シェルをビルド＆書き込み:
   ```bash
   tools/build.sh  src/shell
   tools/upload.sh src/shell
   ```
2. シリアルモニタを開く（115200bps）。バナーとプロンプト `100kinsat>` を確認。
3. `imu init` を実行し「初期化OK」を確認（未検出なら配線/アドレス/電源を確認）。
4. `help` で `land` の書式を確認する。

## 2. 手順A: 据置で静止→着地確定するか

本体を机の上に**静止**させ、`land mon 100` を実行する（約50ms周期で約5秒）。

```
land mon 100
```

各行は `[i/n] |a|=… mean=… var=… still=…ms -> 状態` の形で出る:
```
[ 1/100] |a|=9.79 mean=9.79 var=0.000 still=0ms   -> 計測中（窓充填中）
[25/100] |a|=9.81 mean=9.80 var=0.002 still=0ms   -> 静止中（着地確定待ち）
[45/100] |a|=9.80 mean=9.80 var=0.002 still=1000ms -> 着地検知（静止確定）
```

観察ポイント:
- **据置で `|a|` が 9.8 前後**（重力込み ACC を読めている証拠。0 付近なら誤レジスタ＝gotchas B16 を疑う）。
- **`var` が小さい**（据置なら 0.0x 台）。
- 窓充填（既定20サンプル≒1秒）後に「静止中」→ 静止1秒継続で「**着地検知**」へ。
- 一度「着地検知」になったら以降ラッチ保持される。

### 記録

| 項目 | 期待 | 実際 | 判定 |
|---|---|---|---|
| 据置時 `|a|` | 9.8 前後（重力込み） | 9.66（9.64〜9.70） | OK（重力込み ACC を読めている＝B16回避） |
| 据置時 `var` | 小（0.0x 台） | 0.000 | OK |
| 「着地検知」までの時間 | 窓充填+静止1秒 ≒ 約2秒 | 約 2.2 秒（[43] still=1022ms で確定） | OK |

> 注: 据置中も `land: 加速度読み取り失敗（I2C…）` が約12%出るが（BNO055 クロックストレッチ＝gotchas B17）、
> 失敗サンプルは検知器へ与えず静止時間も計上しない（過少カウント＝安全側）ため着地確定は正しく成立する。
> 原因の切り分けは `imu i2cdiag`（失敗フェーズ内訳／SYS_ERR／方式×クロック A/B）で実施済み（手順C）。

## 3. 手順B: 運動→静止の切り分け（着地衝撃の誤判定防止）

`land mon 200` を実行しながら次を行う:

1. 本体を**手で持って揺らす／振る** → `var` が上がり「運動中」になり、`still` が 0 に戻ることを確認。
2. 机に**コツンと着地衝撃**（軽く落とす/叩く）を与える → スパイクで一瞬 `|a|` が跳ね、「運動中」のまま
   確定しないことを確認（衝撃単発では着地確定しない）。
3. 衝撃後に**そのまま静止**させる → 数秒後に「着地検知」へ至ることを確認（着地→静止の正常系）。

### 記録

| 動作 | 期待 | 実際 | 判定 |
|---|---|---|---|
| 揺らす | `var` 上昇・「運動中」・`still`=0 |  |  |
| 着地衝撃（単発） | スパイクで確定しない |  |  |
| 衝撃後に静置 | 数秒で「着地検知」 |  |  |

## 4. 手順C: 読み取り堅牢性（振動・失敗）

- `land mon` 中に**軽く連続振動**を与え、`land: 加速度読み取り失敗（I2C…）` が出るか（出ても静止判定に
  0 が流れず、その周期はスキップされること）を確認する。
- 走行振動下での失敗率は #17/#18 の走行試験でも観察し、失敗サンプルは検知器へ与えない運用を確認する。
- **I2C 読み取り失敗の原因切り分けは `imu i2cdiag [n]` を使う**。加速度I2C読みを繰り返し、
  失敗率／失敗フェーズ内訳（writeAddr/requestFrom/shortRead）／リトライ回復率／SYS_ERR を集計し、
  さらに repeated-start/STOP × 100k/50k の 4通りを実機A/Bする（`imu init` 後に実行）。

### 記録

| 条件 | 読み取り失敗の有無 | 静止判定への影響 | 所見 |
|---|---|---|---|
| 据置 | あり（約12〜25%） | なし（失敗周期はスキップ・still過少カウントで安全側） | 全失敗が requestFrom 相・SYS_ERR=0 → BNO055 クロックストレッチ（gotchas B17） |
| 振動 | （#17/#18 走行試験で観察） |  | 据置で既に発生するため振動固有ではない |

**`imu i2cdiag 100` 実測（据置）:** 失敗は 100% requestFrom 相・writeAddr=0・SYS_ERR=0・self_test=0x0F。
方式×クロックの A/B は RS100k=24% / STOP100k=28% / RS50k=23% / STOP50k=17% で、n=100 の誤差内で
**有意差なし（方式・クロックのどれでも下がらない）**。3回×5ms の周期内リトライ後も hardFail 7〜9% が残る
（busy窓を跨げないため）。→ 対策は**周期間リトライ（別窓で拾う）で吸収**する設計。read 経路の変更は不要。
なお**運用 I2C クロックは Adafruit begin() が設定する 50kHz**（上表の現行運用点は RS@50kHz=23%。100k 側は
i2cdiag が一時的に強制する実験値で、測定後は 50kHz へ戻す＝gotchas A5/B17）。

## 5. しきい値調整の申し送り

手順A/B で得た **据置ノイズ（`var` の実測レンジ）** と **着地衝撃のスパイク量** をもとに、
`landing::defaultStillConfig()`（`src/lib/core/landing.cpp`）の暫定値を必要に応じて調整する:

- `maxVariance`（分散上限）: 据置 `var` の実測を十分上回り、揺動を弾く値。
- `gTolerance`（g 許容）: 据置 `|a|` の偏り（取付・温度）を吸収する値。
- `requiredStillMs` / `windowSize`: 着地判定の応答性 vs 誤検知耐性のトレードオフ。制御周期に合わせる。

調整結果と実測レンジは #17（ミッションステートマシン）へ申し送る。
