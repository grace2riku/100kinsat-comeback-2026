# パラシュート切り離し（電熱線 D06）実機 bring-up 手順

Issue #13。着地確定後に電熱線（ニクロム線, `D06`→FETゲート→6V直結）を加熱してパラシュート保持部を
溶断する。**安全（過加熱＝発火・やけど・電池消耗の防止）が最優先**。core ロジックは
`src/lib/core/separator`（ホストテスト済 `test/core/test_separator.cpp`）、実機結線は `src/shell` の
`separate` コマンド。

---

## 回路の要点（`doc/cansat_specification/cansat_heat.png` / hardware.md「電熱線回路」）

- `D06`(`HEAT_3v3`) → Q1 2N7000(Nch) → Q2 AO3401A(Pch, ハイサイド) → **J9 のニクロム線** → GND。`D06`=HIGH で加熱。
- **SW2 = ハイサイド電源の直列アーミングスイッチ**。**オープン（ショートピン無し）では +6V が Q2 に届かず、`D06` を HIGH にしても加熱できない**（安全インターロック）。**加熱には SW2 を閉じてアームする**。
- **R25 10k = ゲートプルダウン**（`HEAT_3v3`→GND）。ブート窓で `D06` がハイインピーダンスでも非加熱（B18 を HW で解消）。
- **D4(LED) = `HEAT_3v3` の状態表示**（`D06` HIGH で点灯。ゲート信号なので **SW2 オープンでも点灯**する＝加熱の有無とは別物）。
- 安全3層: ① SW2 直列アーミング ② R25 プルダウン ③ ソフト（早期LOW＋上限ガード＋RAII）。

## ⚠️ 安全上の注意（必読）

- **手順A（ファーム論理確認）は SW2 オープンのまま行う**（電源側が遮断され加熱リスクゼロ）。**手順B（実際の溶断）だけ SW2 を閉じてアーム**し、可燃物のない場所で行う。
- **加熱部（電熱線）は非常に高温になる。通電中・通電直後は絶対に触れない**（やけど）。software.md §5.8。
- **可燃物のそば・密閉容器内で通電しない**。周囲に燃えるものを置かない。消火手段を手元に用意する。
- **最初は短い `heatMs`（例: 300〜500ms）から**始め、様子を見ながら伸ばす。いきなり長時間通電しない。
- **電池（6V, 単4×4）は発熱で消耗する**。連続試験は避け、必要な回数だけ撃つ。
- 席を離れるときは通電しない。異臭・過度な発煙があれば即 `separate stop` かボードリセット・電源遮断。
- 実機の残留リスク: `separate` はソフト上限（`heatMs`／`kMaxHeatMsCap`）で自動停止するが、**MCU がハング
  すると D06 が HIGH のまま**になりうる。長時間の無人通電はしない（恒久対策のハードウォッチドッグ/
  ワンショットは将来課題）。

## 0. 前提

- `tools/upload.sh src/shell <port>` で書き込み、`tools/monitor.sh <port>`（115200bps）で接続。
- 起動時、`setup()` が **D06 を出力＋LOW（非加熱）** に確定する（誤加熱防止）。プロンプト表示直後に
  電熱線が発熱していないことをまず確認する。
- `separate` はコマンド実行中だけ加熱し、**完了・中断・開始不可のどの経路でも最後に D06 を LOW** にする。

## 1. 手順A: ファーム論理の確認（**SW2 オープン＝加熱リスクゼロ**）

**SW2 をオープン（アームしない）のまま**行う。電源側が遮断されているので、`D06` を HIGH にしても加熱しない。
ファーム（HIGH/LOW・自動停止・中断・クランプ）の確認に集中できる。**D4(LED) が `D06` の HIGH で点灯**するので
目視で分かる（SW2 オープンでも点灯する＝これは加熱ではなくゲート信号の表示）。厳密に FET 出力を見たい場合は
J9 端子にテスターを当て、SW2 オープンで**加熱電圧が出ないこと**も確認する。

```
separate 500
separate 999999   （クランプ確認）
```

観察ポイント:
- `D06=HIGH 加熱中 elapsed=…ms` が表示され、**約 500ms で `加熱完了（D06 LOW）`**（D4 LED 点灯→消灯）。
- **短パルスも上振れしない**: `separate 1` でも約100ms 通電し続けず即 LOW（待ちを残り加熱時間でクランプ済み）。
- 途中で**非改行キー**で `中断しました（D06 LOW）`。`separate stop` で LOW（保険）。
- `separate 999999` は `kMaxHeatMsCap`(10s) で必ず停止。
- SW2 オープン中は J9 に加熱電圧が出ない（＝SW2 が電源側インターロックとして効いている）。

| 項目 | 期待 | 実際 | 判定 |
|---|---|---|---|
| **電源投入〜プロンプトのブート中** | **発熱しない**（R25 ゲートプルダウン。gotchas B18） | ブート中〜プロンプト表示前まで D4 消灯（発熱なし） | ✅ |
| 起動直後の D06 | LOW（非加熱・D4 消灯） | 起動直後 D4 消灯 | ✅ |
| SW2 オープンで `separate` | D4 は点灯するが **J9 に加熱電圧が出ない** | `separate 999999` 実行中 D4=2.4V（ゲート信号 HIGH で点灯）だが J9 間=0V（電源側インターロックで加熱電圧なし） | ✅ |
| `separate 999999` | `kMaxHeatMsCap` で自動停止 | `heatMs=10000` に丸め、9750ms まで加熱後 自動 LOW | ✅ |
| `separate 500` の加熱時間 | 約 0.5 秒で自動 LOW | elapsed=250ms 表示後、約0.5秒で `加熱完了（D06 LOW）` | ✅ |
| 非改行キー中断 | 即 LOW | `separate 999999` を 750ms で非改行キー→`中断しました（D06 LOW）` | ✅ |

> ブート窓の注意（B18）: `setup()` は D06 の LOW 初期化を**最初**に置いてソフト側の窓を最小化している。
> 本機は **R25 10kΩ がゲートプルダウン**として実装済みなので、スケッチ起動前のブート期間に D06 が
> ハイインピーダンスでもゲートは GND へ固定され非加熱（HW で B18 を解消）。実機で「電源投入直後
> （プロンプト表示前）に D4 が点灯しない＝非加熱」を念のため確認する。

## 2. 手順B: 溶断時間（`heatMs`）の詰め（**SW2 を閉じてアーム**）

**⚠ ここで初めて SW2 を閉じる（ショートピン挿入＝アーム）**。加熱可能状態になるので、可燃物のない場所で、
消火手段を用意し、実際のパラシュート保持部（テグス等）をセットして、**短い `heatMs` から徐々に上げて**
確実に切れる最小時間を探す。周囲の安全を確保し、1回ずつ確認する。終わったら **SW2 をオープン（ディスアーム）** に戻す。

```
separate 800
separate 1200
separate 2000   （必要に応じて）
```

- 切れた最小 `heatMs` を記録し、余裕（マージン）を持たせた値を運用値とする。
- `kMaxHeatMsCap`（現 10000ms）を超える値を与えても、その上限で必ず止まる（過加熱防止）ことも確認する。

## 3. 手順C: 安全系の確認

- [x] **SW2 オープンで加熱しない**（`separate` 実行しても J9 に加熱電圧が出ない＝電源側インターロック）。※`separate 999999` 中 D4=2.4V／J9=0V を確認
- [x] 起動直後に D06 が LOW（`setup()` の安全初期化・D4 消灯）。
- [x] `heatMs` 到達で**必ず自動 LOW**（上限ガード）。手で押さえ続けなくても止まる。
- [ ] 加熱中に `separate` を重ねて叩いても**多重加熱にならない**（コマンドはブロッキングなので基本起きないが、
      core レベルでも加熱中 `start()` は拒否される）。※ブロッキング仕様のため実機で重ね打ち不可、core ホストテストで担保
- [x] 巨大な `heatMs`（例 `separate 999999`）でも `kMaxHeatMsCap` で止まる。
- [x] `separate stop` と非改行キー中断でいつでも LOW にできる。

## 4. 分離成功確認と再試行ポリシー（検討事項・Issue #13 DoD）

本機には**分離の成否を直接検知するセンサが無い**（リミットスイッチ等は非搭載）。したがって:

- **既定は単発**（`defaultSeparatorConfig().maxAttempts = 1`）。溶断できる `heatMs` を手順B で確保することで
  1 発で切る前提にする。
- core は `maxAttempts` による**有界の再試行**をサポートする（正常完了後のみ再 `start()` 可、`abort()` は端末）。
  フライトのステートマシン（#17）で「分離後に一定時間 走行して機体が動く／GNSS が変化する」等の**間接指標**で
  分離を推定し、動かなければ `maxAttempts` の範囲で再加熱する、という運用を検討する。無限再試行は電池を
  消耗し発火リスクを上げるため**必ず上限を設ける**。
- 直接確認が必要なら将来ハード追加（分離検知スイッチ）を検討（本 Issue の範囲外）。

## 5. 申し送り（#17 ミッションステートマシン / #22 フィールド試験）

- 運用 `heatMs`（手順B の実測＋マージン）と `maxAttempts`（再試行方針）を #17 の分離ステップへ渡す。
- 分離は**着地確定（#12）後**に 1 度だけトリガーし、加熱中は走行を開始しない（順序の担保は #17）。
- **打ち上げ前チェックリストに「SW2 アーム（ショートピン挿入）」を必須項目として入れる**（#22）。運搬・整備中は
  SW2 オープン＝非加熱で保管し、投入直前にアームする。**アーム忘れ＝分離不発**なので運用手順で担保する。
- 実測値は本書の表を埋めてコミットする。
