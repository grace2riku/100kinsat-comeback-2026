# 放出(上空放出)検知 立ち上げ・キャリブレーション手順（CdS照度センサ A0 / Issue #11）

ホストテストで確認できるのは「**HW非依存ロジック**」だけ（gotchas A）。具体的には照度生値から
放出（暗→明）を判定する状態機械（`test/core/test_release_detect.cpp`：`ReleaseDetector` /
`isReleasedLevel` / `isReleasedNow` / `hasReleased`、極性・ヒステリシス・継続時間）。次の項目は
実機でしか確認できない:

- **A0 のアナログ生値取得**（`analogRead(A0)` が装置内(暗)／上空相当(明)で十分な差を出すか）
- **しきい値・極性の妥当性**（装置内と上空の生値の中点付近に閾値が置けるか。分圧の向き＝Brighter/Darker）
- **暗→明の継続で放出確定へ収束**（明所へ出して数百 ms 後に「放出検知」へ至るか）
- **誤検知しない堅牢性**（雲・影・機構の陰による揺れやノイズを継続時間／ヒステリシスで弾けるか）

本書はその計測手順と記録テンプレート、および**閾値キャリブレーション手順**（Issue #11 DoD）を兼ねる。
計測値を埋めてコミットし、Issue #11 のHW依存DoDを満たしたうえでマージする（HWモジュールのマージ
ゲート: 実機確認まで Issue をクローズしない）。

対象コマンド: シェル（`src/shell`）の `cds [mon [n]]`
判定ロジックは `src/lib/core/release_detect`、生値取得は `.ino` 内 `analogRead(A0)`（HAL 薄呼び出し）。

## 0. ⚠️ 注意・前提

- **用語「装置内」**: ロケットに搭載された **CanSat 収納スペース（放出機構・キャリア）の内部**のこと。
  打上げから上空放出までの間、CanSat はこの中に収まっており**遮光されてほぼ暗黒**になる。放出検知は
  この「装置内（暗）→ 上空（明）」の照度差を使う。地上試験では**遮光した箱・袋・黒布**で模擬する
  （手順A'）。
- **タイマ起動を避ける（ルール推奨）**: 放出開始は「打上げから N 秒後に自動で走り出す」（タイマ起動）
  ではなく、本モジュールの**照度センサによる実測検知**で行う（大会要領 別紙1 CanSat部門共通）。
  `requiredMs` は「放出側の明るさが連続したことの確認時間」であり、時間経過そのものを放出とはしない。
- **電源 ON は装置内（暗）で行う**。検知器は**アーミング**（暗=OFF を一度観測してからのみラッチ許可）
  を持つため、起動時に既に明るいままだと**確定しない**（`cds mon` は `放出側だが未アーム` と表示）。
  これは電源投入時点灯・断線 stuck-at-bright の誤ラッチを弾く安全機構だが、**放出前は必ず暗い装置内に
  収めて電源 ON**し、一度暗（アーム）を経てから明へ遷移させる運用が前提（起点は最初に暗を見た時点）。
- **分圧の向きに依存**（software.md §5.2）: CdS は「明→生値が小さい／暗→生値が大きい」のどちらにも
  なりうる。本手順の**手順A**で明暗の生値を測り、明の方が**小さい**なら `Polarity::Brighter`（既定）、
  明の方が**大きい**なら `Polarity::Darker` を選ぶ。
- **断線・ショート時の張り付き**に注意: 配線断で生値が 0 や 1023 に張り付くと、極性次第で「明るい」と
  誤認しうる（Brighter で 0 張り付き＝放出側）。放出後は不可逆（ラッチ）なので、配線・はんだを事前確認する。
- **`cds` / `cds mon` は読み取り専用で HW を駆動しない**ため安全（暴走しない。gotchas B5）。`mon` 中は
  **非改行キー入力で即中断**できる（CR/LF はコマンド確定の残留改行なので中断トリガーにしない。gotchas B10）。
- **しきい値は実機校正済み（室内代替測定）**（`release_detect::defaultReleaseConfig()`：閾値960・
  ヒステリシス10・Brighter・0.5s継続。2026-07-18 手順A' 実測 D=1022 / B=896 より）。屋外昼間での
  再校正は #22。#17（ステートマシン）/#14（ロギング開始トリガ）へ申し送る。

## 2. 閾値キャリブレーション手順（Issue #11 DoD）

放出検知の3設定（`threshold` / `hysteresis` / `polarity`）を実機生値から決める。

### 手順A: 明暗の生値を測る（極性と閾値の材料）

1. **装置内（暗）** に CanSat を収めた状態で `cds` を数回叩き、生値を記録する（＝暗時生値 **D**）。
2. **上空相当（明）** ＝装置から出し、屋外/明所（放出後に想定される明るさ）で `cds` を数回読み、
   生値を記録する（＝明時生値 **B**）。曇天・日陰など**最も暗い放出後条件**でも測ると安全側。

```
cds            （生値を1回表示。数回叩いて安定値を見る）
```

観察ポイント / 記入:

| 条件 | 生値（複数回） | 代表値 |
|---|---|---|
| 装置内（暗） D | 1022, 1022, 1022, 1022, 1022（`cds mon` 100回も全て 1022） | **1022** |
| 上空相当（明・最悪条件）B | 894, 894, 894, 896, 895（`cds mon` 100回は 894..897） | **896** ※室内照明（手順A'）。屋外昼光は #22 で再測定 |

### 手順A': 夜間・室内での代替測定（作業時間が夜間になる場合）

屋外昼光が使えない夜間・室内でも、**極性判定とロジック確認（手順C/D）は完結できる**。検知器の動作
確認に必要なのは「暗と明で生値に十分な差が出ること」であり、絶対照度ではないため、以下で代替する:

| 本来の条件 | 夜間・室内での代替 |
|---|---|
| 装置内（暗）D | 段ボール箱・遮光袋に入れる／CdS を黒布や手で完全に覆う |
| 上空相当（明）B | 部屋の照明を点け、CdS を天井照明へ向ける（拡散光での安定値を使う） |

- 室内照明（数百 lx 程度）は昼光（屋外は曇天でも数千〜1万 lx 超）より**暗い**。つまり「室内の明 B」で
  暗と分離できる閾値なら、実際の放出後（昼光）はさらに明側へ振れるため、**Brighter 極性では安全側**に
  働く（暗判定＝装置内の判定はそのまま有効）。
- ただし **threshold / hysteresis の最終値は屋外昼間で再測定して確定**する。特に手順Aの「曇天・日陰の
  最悪条件 B」は屋外でしか測れない（申し送り: #22 屋外試験の DoD）。夜間室内で決めた値は**暫定値**として
  `defaultReleaseConfig()` に反映してよい。
- スマホライト・デスクライト等の**点光源**は当てる角度・距離で生値が大きく揺れるため、閾値決定には
  使わない。代わりに**手順D の「一瞬だけ強い光」（単発閃光を弾く確認）に流用**すると夜間でも手順Dが
  完結する。
- 手順C（暗→明遷移）は「箱から出して天井照明へ向ける」、手順D の明所起動は「部屋の照明下で
  `cds mon` を開始」で夜間室内でもそのまま実施できる。

### 手順B: 設定を決める

- **極性**: `B < D`（明の方が小さい）なら `Polarity::Brighter`（既定）。`B > D` なら `Polarity::Darker`。
- **threshold**: 暗と明の**中点** `(D + B) / 2` を起点に、放出後条件（明側）へ確実に振れる値へ寄せる。
  D と B の差が小さい（分離不良）なら遮光・センサ位置を見直す（十分な明暗差が前提）。
- **hysteresis**: `|D - B|` の 5–10% 程度（閾値付近の微振動幅より大きく、明暗差より十分小さく）。
- **requiredMs**: 放出後すぐ検知したいが、閃光/影の単発は弾きたい。既定 500ms を起点に実機で調整。

決めた値は `defaultReleaseConfig()` を更新する（本 Issue のスコープ。#17 で最終確定）。

## 3. 手順C: 暗→明の遷移で放出確定を観測

```
cds mon        （既定100回・約50ms周期＝約5秒観測）
cds mon 300    （回数指定。300回≈15秒。手順Cは持ち替え時間を含むためこちらを推奨）
```

> ⚠️ **暗→明の遷移は1回の `cds mon` セッション中に行うこと**。`cds mon` は開始のたびに検知器を
> `reset()` する（各観測を独立させるため）ので、「暗で1回完走→明所で再実行」と分けると、2回目は
> そのセッション内で暗を観測しておらず**未アームのまま確定しない**（これは手順Dの明所起動チェック
> と同じ挙動＝アーミングの正しい動作）。既定の100回（約5秒）では持ち替えが忙しいので `cds mon 300`
> （約15秒）を使う。

1. **装置内（暗）** から始め、表示が `-> 放出前（暗）` であることを確認。
2. **同じ `cds mon` 実行中に** **明所へ出す**（箱から出す・遮光を外す）。表示が
   `-> 放出側（確定待ち）`（`elapsed` が増える）→ `requiredMs` 到達で `-> 放出検知（確定）` へ進む。
3. 一度 `放出検知（確定）` になったら、**同じセッションの残り回数の間に**再び暗くし（箱へ戻し）、
   **ラッチ保持**（`放出前` に戻らず `放出検知（確定）` のまま）であることを確認。

期待表示例（Brighter・閾値960・実測値ベースの場合）:

```
[12/100] cds(A0)=1022 elapsed=0ms  -> 放出前（暗）
[13/100] cds(A0)=896 elapsed=0ms  -> 放出側（確定待ち）
[14/100] cds(A0)=895 elapsed=50ms  -> 放出側（確定待ち）
 ...
[22/100] cds(A0)=896 elapsed=500ms  -> 放出検知（確定）
```

## 4. 手順D: 誤検知しないこと（チャタリング／単発）

- **閾値付近でゆっくり手をかざす**など、明暗の**境界で揺らす**。`確定待ち`↔`放出前` を行き来し、
  `elapsed` が積み上がらず **`放出検知` に至らない**ことを確認（継続時間リセット＋ヒステリシス）。
- **一瞬だけ強い光**（フラッシュ・反射）を当て、単発では確定しないことを確認。
- **明所で `cds mon` を開始**（暗を観測しない＝電源投入時点灯の模擬）し、`放出側だが未アーム` と
  表示され **`放出検知` に至らない**ことを確認（アーミング＝暗→明遷移の要求。断線 stuck-at-bright 対策）。

### 実機確認結果（2026-07-18 手順A/B：夜間・室内の代替測定＝手順A'）

- **測定条件**: 夜間・室内。暗＝段ボール箱（遮光）、明＝部屋の照明を点け CdS を天井照明へ向ける。
- **実測生値**: 装置内（暗）D = **1022**（`cds` 5回・`cds mon` 100回とも 1022 で安定）。
  室内照明（明）B = **894..897**（代表値 **896**。ノイズ幅 ±2 程度で安定）。
- **極性**: B < D（明の方が小さい）→ **`Polarity::Brighter`**（既定どおり）で確定。
- **決定した設定**（`defaultReleaseConfig()` へ反映済み）:
  - `threshold = 960`（中点 (1022+896)/2 ≈ 959。屋外昼光はさらに明側へ振れるため室内基準は安全側）
  - `hysteresis = 10`（|D−B|≈126 の約8%。OFF 復帰しきい値 970 で暗 1022 と十分分離）
  - `polarity = Brighter` / `requiredMs = 500.0`（既定を維持。手順C/D で検証）
- **補足**: B は室内照明の代替測定（手順A'）。屋外昼間（曇天・日陰の最悪条件）での再校正は
  #22 の DoD に申し送り済み。
- **手順D（明所起動）2026-07-18 実施**: 校正値反映後のファームで、明所（894前後）のまま `cds mon` を
  開始。暗を観測していないため elapsed が 5050ms まで積み上がっても `放出側だが未アーム` のまま
  **確定しない**ことを確認（アーミング安全機構が実機で有効。断線 stuck-at-bright / 明所起動対策）。
  暗（1022）では `放出前（暗）` 表示・elapsed=0 で安定。
- **手順C 2026-07-18 実施（合格）**: `cds mon 300` の単一セッション内で
  暗 1022（アーム）→ 箱出し 954..884（`確定待ち`・elapsed 積算）→ **elapsed=500ms ちょうどで
  `放出検知（確定）`** → 箱へ戻し 1022 でも **確定表示を約6.7秒保持（ラッチ）** を確認。
  戻し途中の 948..966（閾値960±ヒステリシス帯付近）でも表示は乱れず、ヒステリシス帯の据え置きも
  実機で確認できた。
- **残り**: 手順D の残り2項目（閾値付近の揺らぎで確定しない・単発閃光で確定しない）。

## 6. 申し送り（#14 ロギング / #17 ステートマシン）

- **ロギング開始トリガ**: 大会要領は「放出後〜動作停止まで」の制御履歴を要求。#17 で本モジュールの
  `hasReleased()` を**ロギング開始（#14）／降下フェーズ遷移のトリガ**に結線する（datalog_bringup.md §4 の
  申し送りと対応）。
- **検知後の状態遷移**: `hasReleased()` を `INIT → DESCENT` の遷移条件にし、放出前は走行を開始しない
  （放出検知＝ミッション開始トリガ、#17 の `MissionState`）。
- **読み取り周期**: 本コマンドは約50ms周期。飛行時の実周期（制御ループ）に合わせて `requiredMs` の
  妥当性を #17/#18 で再確認する（周期が変わると継続時間の意味が変わる）。
